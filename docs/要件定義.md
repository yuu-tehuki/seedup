# 要件定義 - Seedup

## サービス概要

**サービス名**: Seedup
**概要**: デジタル系起業家と応援者をつなぐクラウドファンディングプラットフォーム

---

## ターゲットユーザー

| ユーザー種別 | 説明 |
|---|---|
| 起業家（プロジェクトオーナー） | デジタル領域でアイデアや事業を立ち上げたい個人・チーム |
| 応援者（バッカー） | 面白いプロジェクトを発見・支援したい個人 |

---

## MVP機能

### 1. ユーザー登録・認証
- メールアドレス＋パスワードによる新規登録
- ログイン・ログアウト
- プロフィール情報の設定（表示名、自己紹介）

### 2. プロジェクト投稿
- プロジェクトの新規作成（タイトル・説明・目標金額・期限・カテゴリ・サムネイル画像）
- プロジェクトの編集・削除（オーナーのみ）
- プロジェクト一覧・詳細ページの表示

### 3. 支援機能
- プロジェクトへの金額指定による支援
- 支援履歴の確認（バッカー側）
- 支援状況・達成率の表示（プロジェクト詳細ページ）

### 4. 進捗報告
- オーナーによる進捗アップデートの投稿
- 支援者向けアップデート一覧の表示

---

## 技術スタック

| 項目 | 技術 |
|---|---|
| フロントエンド | Next.js (App Router) |
| データベース | Supabase (PostgreSQL) |
| 認証 | Supabase Auth |
| ストレージ | Supabase Storage（画像アップロード） |
| デプロイ | Vercel |
| スタイリング | Tailwind CSS |

---

## データモデル（概要）

### users（Supabase Auth + プロフィール）
- id, display_name, bio, avatar_url, created_at

### projects
- id, owner_id, title, description, goal_amount, current_amount, deadline, category, thumbnail_url, status, created_at

### pledges（支援）
- id, project_id, backer_id, amount, created_at

### updates（進捗報告）
- id, project_id, title, body, created_at

---

## 非機能要件

- レスポンシブデザイン対応（モバイル・PC）
- Supabase Row Level Security (RLS) によるデータアクセス制御
- 環境変数による機密情報の管理
